# Stage 5 — Report (substance gates, verdict, publish)

The verdict is computed deterministically by `scripts/aggregate_verdicts.py` from the
fleet's structured outputs — there is **no per-case 0–100 confidence score**. The report
consumes `verdict-final.json`; it never re-derives the label.

## Inputs the aggregator reads

- plan frontmatter (`tier`, `channel`, `journeys`, `changed_surfaces`),
- every `persona-*/findings.json` (`value_verdict` ∈ delivered|partial|failed,
  `sad_paths_walked`, `bugs[]`, `ux_friction[]`, `coverage`),
- `verifier/confirmations.json` (adversarial cold-repro results),
- `coverage-critic.json` (`unreached_changed_surfaces`, frontier).

Label precedence: `blocked` (degraded channel / unreached PRIMARY surface / Tier-3 env
gate) → `ai-rejected` (≥1 confirmed blocker) → `ai-needs-human` → `ai-approved`.
High-severity UX friction never rejects by itself but pushes toward `ai-needs-human`.

## REVIEW.md required sections (in order)

1. **Plain-language opener** — ONE sentence a non-engineer understands ("In clean words:
   the new sysadmin pages work, but approving a chat proposal twice creates two
   companies — don't ship until that's fixed."). The verdict label comes AFTER the
   sentence, never instead of it.
2. **Requirements summary** — AC in your own words, with per-AC source attribution.
3. **`## Context gaps`** — the Stage 1 audit trail (questions+justifications, or
   `none — sufficient from <sources>`).
4. **`## Journey map`** — journeys, tier (computed/emphasis/chosen + rationale),
   personas, **execution mode** (parallel fleet / inline sequential + why).
5. **Per-journey value verdicts** — delivered|partial|failed, evidence artifact, sad
   paths walked, one-line rationale each.
6. **`## Regression sweep`** — one row per `regression-checks.json` entry:
   `REG-id — walked|not-applicable(<reason>) — pass|FAIL — artifact`. Required whenever
   the ledger matched; "no ledger entries matched this diff" otherwise.
7. **`## Sad-path attestation`** — per user-facing AC; Tier ≥2 must be satisfied.
8. **`## Coverage frontier`** — what was NOT reached. Empty frontier on a non-trivial
   diff is flagged as suspicious, not celebrated.
9. **`## Confirmed blockers`** / **`## Unconfirmed findings`** — confirmed = has a
   verifier artifact; unconfirmed never gates but is never buried.
10. **`## UX friction`** — journey, note, severity.
11. **`## Decision needed`** + **`## Frontier — how to close`** — MANDATORY when label ∈
    {ai-needs-human, ai-rejected}: the single decision the human must make, and per
    frontier item `{item — why_unreached — how_to_close}`. A bare label with neither
    block is a report-gate FAILURE.
12. **Findings to act on** — for each confirmed/unconfirmed finding: severity,
    file:line, copy-pasteable repro, suggested next action (the fix-mode work order).
13. **Environment & heals** — env_realism summary, any self-heals applied
    (`environment.md` §5), execution-mode degradations, autostash ref if any.
14. **Blockers / Deployment notes / Limitations** — filled or explicit `None`.
15. Footer: `*Generated by AI business review (v3) — br-sha:<SHORT_SHA>*`.

Update `.context/reviews/local/LATEST.txt` (local mode) with `<REVIEW_DIR>  <label>`.
PR mode: write the same line into `$REVIEW_DIR/LATEST.txt` AND the cross-run pointer
`.context/reviews/LATEST.txt` (`<REVIEW_DIR>  <label>  <iso-date>`) so "what was the
last review?" has one stable answer regardless of mode.

## Gate 6b — substance (each verified against verdict-final.json, not prose)

1. Journey map present + non-empty for a user-facing diff (or `no_surface_reason`).
2. Per-journey verdict + evidence artifact for every touched journey.
3. Sad-path attestation present; Tier ≥2 satisfied — else the label cannot be
   `ai-approved`.
4. Coverage frontier present; empty-on-broad-diff flagged.
5. Every confirmed blocker has a `verifier/confirmations.json` entry `confirmed: true`
   + verifier artifact.
6. **Capability attestation**: every browser-derived claim traces to a real artifact
   (screenshot / snapshot_diff / a11y_ref); code-trace-only claims stamped
   `NOT browser-verified`. Zero browser artifacts on a user-facing diff → `blocked`.
7. **Regression sweep section present and consistent with `regression-checks.json`** —
   a matched, unwalked, un-waived entry caps the label at `ai-needs-human`
   (enforced by `check_regressions.py --plan` + this gate).
8. Decision-needed gate (section 11) for needs-human/rejected.
9. No placeholders, no `/tmp/` paths, no unresolved `<N>` tokens.
10. **Coverage-critic attestation**: `verdict-final.json.critic_missing_at_tier2plus`
    is false, OR REVIEW.md justifies the skip in one sentence. The critic is how
    "what did the fleet miss?" gets an independent answer — a tier ≥ 2 run that
    silently skipped it is a faked-breadth smell (gap found 2026-06-12).

## Gate 6c — screenshots can NEVER pass vacuously

6c.0: user-facing diff + zero screenshots/browser artifacts → **FAIL** (→ blocked).
6c.1 per artifact: red callout visible around the proving element; literal evidence text
legible inside it; nothing critical clipped; no stale callouts; filename matches the
journey/case. Fix or retake; no exceptions. (Capture sequence: invoke the global
`screenshot-with-callout` skill per shot — `screenshot-rules.md`.)

## The `blocked` path

Render which gate/preflight step failed and that NO trustworthy live test happened;
stamp code-trace glimpses `NOT browser-verified`; escalate retry-now / fix-env-and-resume
(with the concrete `how_to_close` per failed gate) / abort; STOP. No verdict label, no
publish, no GitHub label, no end-of-run prompt.

## Publish (healthy runs only; opt-in)

| Invocation | Behavior |
|---|---|
| `--publish` (PR mode) | post immediately after gates pass |
| default | end-of-run prompt (below) |
| `--no-prompt` | print `Report: $REVIEW_DIR/REVIEW.md`, exit |
| local mode | offer publish only if the user supplies a PR number at the prompt |

**Screenshots are ALWAYS inline in the posted comment** (field rule: asked twice, then
demanded). Check repo visibility first (`gh repo view <owner>/<repo> --json visibility`)
— raw-URL images only render inline for PUBLIC repos. Two supported mechanisms — try in
order:
1. **`gh attach`** (the `Addono/gh-attach` extension; verified working 2026-06-12,
   PR 336 — bare `upload <png>` errors with "Target is required"):
   ```bash
   gh attach upload --quiet --target "relaticle/relaticle#<N>" \
     --strategy repo-branch --format url <png>   # prints a hosted raw URL
   ```
   `repo-branch` commits the file to a `gh-attach-assets` branch and returns a
   `https://github.com/<owner>/<repo>/raw/refs/heads/gh-attach-assets/<uuid>/<file>`
   URL — curl it for 200 before embedding.
2. **evidence branch** (fallback): `git checkout --orphan review-evidence-pr-<N> && git add <pngs>
   && git commit && git push origin review-evidence-pr-<N>`, reference raw URLs
   (`https://raw.githubusercontent.com/relaticle/relaticle/review-evidence-pr-<N>/<path>`),
   then return to the review branch. Never post local file paths.

Then: `gh pr comment <N> --repo relaticle/relaticle -F REVIEW.md` (with image refs
rewritten), remove old `ai-*` labels, add the new one, save the comment id. Idempotency:
a comment ending `br-sha:<SHORT_SHA>` for the same head SHA means already-reviewed —
skip at Stage 1, and **re-check the head SHA right before posting** (a moved head =
stale verdict; re-run or say so in the comment). `blocked` is never a label.

## End-of-run prompt (healthy, no flags, not --no-prompt)

```
========== BUSINESS REVIEW COMPLETE ==========
<plain-language opener sentence>
Verdict: <label>   Tier: <tier>   Channel: healthy   Mode: <fleet|inline>
Journeys: <id:verdict ...>   Confirmed: <n>  Unconfirmed: <n>  UX friction: <n>
Regression sweep: <n>/<n> pass   Frontier items: <n>
Report: <REVIEW_DIR>/REVIEW.md

Options: [1] publish to PR  [2] enter fix mode (fix → re-verify → re-gate)  [N] exit
==============================================
```

When label ∈ {ai-needs-human, ai-rejected}, the prompt ALSO surfaces `decision_needed`
verbatim with close-it-now / human-will-verify / accept-frontier options — a needs-human
label must end as a conversation, not a tag (field rule: "AI need to stop and ask human
decision").
